# Software Architecture Method

Use this sequence:

1. Clarify domain and business outcome.
2. Identify use cases and actors.
3. Identify quality attributes.
4. Identify boundaries and ownership.
5. Choose architecture style only when it solves a real force.
6. Define APIs, events, data ownership, and deployment constraints.
7. Identify risks, trade-offs, and decision needs.
8. Produce ADRs or next actions.

## Evaluation Criteria

- coupling and cohesion
- dependency direction
- testability
- operability
- security handoff
- data ownership
- maintainability
- change frequency
- team ownership
- business value
