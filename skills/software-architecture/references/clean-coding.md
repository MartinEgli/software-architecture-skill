# Clean Coding

Use Clean Coding for code-level architecture hygiene and maintainability review.

## Checks

- Names reveal intent.
- Functions and modules have focused responsibilities.
- Duplication is removed when it hides a concept.
- Error handling is explicit and testable.
- Side effects are visible.
- Tests cover important behavior.
- Dependencies are clear and deliberate.
- Comments explain why, not obvious what.
- Public APIs are small and stable.
- Code structure matches domain or use case, not only technical layers.

## Review Output

For each finding include:

- location
- issue
- impact
- recommendation
- test need

Do not rewrite code style for preference only. Prioritize behavior, clarity,
maintainability, and risk.
