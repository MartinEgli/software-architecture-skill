# Contributing

## Workflow

1. Create a branch.
2. Make focused changes.
3. Run validation and tests.
4. Open a pull request using the template.

## Checks

```powershell
npm run validate
npm run test
```

## Skill Changes

Behavior changes must update:

- `skills/example-skill/SKILL.md`
- relevant references
- at least one example when output behavior changes

## Style

- Use precise language.
- Keep rules testable where possible.
- Avoid placeholders in committed files.
- Avoid domain-specific content in this generic template.
